# Teaching Style

> 这是学习内容生成的教学风格配置。后续生成 `lesson.md`、`exercises.md`、`answers.md` 时，优先遵守这里的规则。

## 主参考策略

- 主知识主线只保留：`《Python基础教程（第3版）》`
- 讲解风格基准参考：廖雪峰 Python 教程
- 目标不是百科式整理，而是尽量接近“教程正文”的学习体验

## 内容组织规则

- 先决定“今天学什么”，以学习大纲和主参考书章节为准
- `study/dayN/` 负责当天学习顺序和任务量，真正的知识来源由对应 unit 提供
- 再参考廖雪峰教程的讲法组织内容：
  - 先讲概念是什么
  - 再给最短示例
  - 再补一个需要注意的点
  - 再给一个贴近当前学习主线的变化例子
- 最后再追加增强内容
- 默认把同一主线下 `2-4` 个强相关知识点打包成一天，而不是把知识点切得过碎
- 对有编程基础的学习者，每天应形成一个“小专题”，而不是单个零散概念

## 允许的增强内容

每个学习日默认只增加这三类增强：

- `JS / TS 桥接`
- `更贴近工程的案例`
- `更有强度的练习题`

除这三类外，不要默认加入大量管理型提示。

## lesson 风格要求

- 要像教程正文，不要像任务说明书
- 要有知识解释，不能只剩标题和行动点
- 默认面向“有编程基础的前端开发者”
- 强度要高于零基础入门教材，但仍然保持可读
- 对当前主题的关键概念，要补“它是什么、为什么需要、和相近概念有什么区别”
- 单日 lesson 默认至少覆盖：
  - `1` 个核心概念点
  - `1` 个接口 / 数据结构 / 参数设计点
  - `1` 个真实脚本组织或应用点
- 如果当前主线在“函数 / 脚本 / 数据处理”阶段，不能只讲单个语法点

## exercises 风格要求

- 至少包含：
  - 概念分析题
  - 实现题
  - 改错 / 重构题
  - 边界输入题
- 至少一题贴近当前脚本或项目场景
- 日常练习默认做成一组完整训练，任务量不少于 `5` 组
- 至少有一题要求把多个知识点整合进同一个主流程

## answers 风格要求

- 不只给结果
- 要解释为什么这样设计更好
- 要指出原写法差在哪里

## 版权边界

- 可以参考教程的结构、知识顺序和讲法
- 不要大段照搬在线教程正文
- 如需保留原讲法，只做最小必要改写，并结合本项目的案例和练习
