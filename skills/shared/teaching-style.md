# Teaching Style

> 共享教学风格：所有学习内容默认遵守。

## 主参考策略

- 主知识主线只保留：`《Python基础教程（第3版）》`
- 讲解风格基准参考：廖雪峰 Python 教程
- 目标不是百科式整理，而是接近“教程正文 + 工程练习”的学习体验

## 组织方式

- 先决定今天学什么，再按固定大纲和动态大纲收敛范围
- `study/<day-id>/` 负责当天学习顺序和任务量
- `study/units/` 只作可选索引，不承担当天任务输出
- 默认把同一主线下 `2-4` 个强相关知识点打包成一天

## lesson 风格

- 面向“有编程基础的前端开发者”
- 要像教程正文，不要像任务说明书
- 要讲清：
  - 它是什么
  - 为什么需要
  - 和相近概念有什么区别
- 默认至少覆盖：
  - `1` 个核心概念点
  - `1` 个接口 / 数据结构 / 参数设计点
  - `1` 个真实脚本组织或应用点

## 允许的增强

- `JS / TS` 桥接
- 更贴近工程的案例
- 更有强度的练习题

## 版权边界

- 可以参考教程的结构、知识顺序和讲法
- 不要大段照搬在线教程正文
- 必须结合本项目的 day-first 输出结构和练习场景做最小改写
