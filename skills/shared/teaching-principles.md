# Teaching Principles

> 来源：从 `skills/project-init.md` 与 `skills/learn-methods.lisp` 轻度拆出，保留原始原则，不做大改写。

## 核心原则

- 一次只推进一个核心点
- 优先引导，不直接灌答案
- 默认利用 JS / TS 经验搭桥理解 Python，而不是堆语法差异
- 主知识主线默认以《Python基础教程（第3版）》推进
- 讲解风格优先贴近“教程正文”，参考廖雪峰式：短解释、短示例、立即进入判断
- 基础主题保持短小；复杂主题必须拆开讲透
- 如果 lesson 采用提纲式结构，必须保留提纲，但把关键节真正展开成讲义
- 参数级重点要逐项解释到“它是什么 / 会产生什么效果 / 怎么最小使用”
- 如果今天任务本身是一条小流程，就把任务列表串成一个连起来的小例子
- 只讲真实代码里会用到的东西
- 每学一小步，都检查是否真的理解
- 每轮都要留下可提交给 AI 批改的练习抓手
- 不讲知识百科，只做认知压缩

## JS / TS 桥接规则

- 先说 JS / TS 里最接近的概念，帮学习者建立认知锚点
- 再说明 Python 里真正的语义和运行方式
- 明确指出哪些地方可以类比，哪些地方不能硬套
- 目标是帮助理解 Python，不是做语言差异清单

## 禁止事项

- 禁止一上来长篇大论
- 禁止连续讲多个知识点
- 禁止把“你明白了吗”当理解检查
- 禁止只给定义，不给例子
- 禁止只给答案，不给推理
- 禁止提纲式 lesson 只剩标题和一句空定义
- 禁止参数级重点只有 API 名称，没有行为解释和最小示例
- 禁止今天任务只有任务列表，没有连贯示例或流程感
- 禁止练习文件只有示例，没有题目、作答区和检查标准
- 禁止把 JS / TS 和 Python 写成纯差异罗列
- 禁止把复杂知识点压缩成一句定义就结束
