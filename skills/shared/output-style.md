# Output Style

> 共享规则：所有学习内容默认遵守以下输出风格。

## 规则

- 面向“前端转 Python / AI Agent”
- 优先用 JS / TS 类比帮助建立迁移理解
- 每次讲解都要补充真实用法，不只写定义
- lesson 默认按 `skills/shared/teaching-style.md` 的教学风格生成
- `study/dayN/` 是单日执行视图，`study/units/` 是知识内容视图，不要混用目录职责
- lesson 要写成“高密度教程正文”，不是提纲，也不是提示清单
- 概念必须讲清：它是什么、为什么需要、和相近概念有什么区别
- 涉及函数或方法时，补齐关键参数、返回值、常见误解
- 默认按“有编程基础的人”控制难度，避免只有热身题
- 每天至少给一个真实代码变化例子和一个分析 / 改写任务
- 每天默认覆盖 `3-5` 个强相关知识点，不要只讲一个过窄点
- 当前固定大纲节点如果天然包含多个相邻子点，优先打包在同一天推进
- `practice.py` 默认按“小型训练包”生成，而不是零散 TODO 列表
- lesson 和 practice 的强度要能让有编程基础的学习者完成 `1-2` 小时的学习与动手

## 禁止项

- 空泛总结
- 只列概念，不给例子
- 只写管理性提示，不给有效知识讲解
- 把 lesson 写成大段规则说明书
- 忽略边界情况和报错场景
- 把当天内容收缩成单一语法点的低强度讲解
- 只给 1 到 2 个过于简单的练习任务
