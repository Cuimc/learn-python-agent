# Practice Review Rules

> 来源：主要从 `skills/learn-methods.lisp` 与 `skills/learn-plan.lisp` 轻度拆出。

## 练习文件要求

- 必须同时包含：示例区、题目区、作答区、检查标准
- 文件开头必须列出：本文件会用到的新内容清单
- 至少包含 3 类练习：热身题、编码题、排错题
- 编码题必须预留 `TODO` / `START` / `END` 等清晰作答边界
- 每个练习都要说明目标，不允许只给例子不留任务
- 练习中禁止出现未在 lesson 或补洞里声明过的 API / 方法 / 内置函数
- 必须告诉学习者把什么信息发给 AI 以便批改和纠错

## AI 批改输入

- 题目名称
- 我写的代码
- 我的运行命令
- 我的运行结果或报错
- 我自己的理解
- 我卡住的点
- 我希望 AI 重点检查的地方

## 反馈规则

- 先判断学习者是概念没懂、步骤不会、表达含糊，还是粗心导致
- 先让学习者说自己的理解，再指出缺口
- 先给最小提示，再给半步提示，最后才给完整答案
- 指出错因时，要解释为什么错、以后怎么避免
- 如果是代码题，优先沿用学习者原始写法做最小修正，而不是整段重写

## 排错规则

- 先定位报错类型
- 让学习者先判断问题可能在哪
- 提示先看哪一行、哪个变量、哪个调用
- 引导学习者自己修
- 最后才给修正版
- 解释为什么会错、以后怎么避免
